 \section Сборка программ (build tools)
 
 Сборочная система программы состоит из рекурсивной структуры make-файлов Makepp версии 2.0 (http://makepp.sourceforge.net/).
 Рекурсивная система предусматривает наличие специальных make-файлов для отдельных подчастей программы.
 Сборка каждой программы осуществляется выполнением команды makepp в корневой директории суб-проекта с опциональным заданием переменных BUILD_PORT=xxx BUILD_MODE=xxx (см. описание значений в файле, по умолчанию используется релизная сборка под целевое устройство), а также других переменных, управляющих development-вариантами сборки. Сборка осуществляется в отдельной директории BUILD_<имя_порта>_<конф_сборки> автоматически.
 Сборка поддерживается (теоретически) на всех хостах с интерпретатором perl, но тестировалась только на Windows 7 x64 с установленным ActivePerl 5.16 в среде mSYS 1.0.18.
 Сборка не поддерживается традиционными утилитами make.
 Дополнительно возможна интеграция в С/C++ Project среды Eclipse CDT путем создания Makefile project with existing code и изменением Build command в свойствах проекта c "make" на "makepp" с опциональным заданием переменных и опций, например: "makepp BUILD_PORT=target-device-rev1 BUILD_MODE=debug".
 (Замечание для windows. Необходимо запусать eclipse из среды msys, в системных путях должен быть прописан путь папки bin/ arm gcc тулчейна, в Build command вместо "makepp" надо указывать "perl /usr/local/bin/makepp")
